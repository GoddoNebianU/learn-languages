# 自适应阅读：精准匹配 90%+ 词汇覆盖率的文本推荐

> 状态：设想阶段

---

## 问题

语言学习最有效的方式之一是**大量泛读**——找略高于自己水平的文本，保持"能读懂 90%+"的甜蜜区。但学习者最大的痛点是：

1. 不知道自己认识多少词
2. 找不到恰好匹配自己水平的文本
3. 现有分级读物（如 Graded Readers）粒度太粗，只有几个固定级别

**目标**：为每个学习者实时计算个人词汇画像，从内容库中精准推荐"你能读懂 90%+ 的文本"。

---

## 理论基础

### 词汇覆盖率

一篇文本的"可理解度"可以用**词汇覆盖率**衡量——你认识的词占全文去重词汇的比例。但这里的"词"必须归一化为原形（lemma）：认识 "go" 意味着 "goes/went/gone" 都算认识。同时应排除专有名词和数字，它们不影响理解。

### 为什么是 90%

- **90-95%**：能猜出剩余词意，阅读流畅，有成就感。这是"可理解输入"（Krashen 的 i+1）的理想区间。
- **80-89%**：频繁卡顿，需要大量查词，挫败感强。多数学习者在这个区间放弃。
- **>98%**：太简单，学不到新东西，浪费时间。

### 词频分布比单一覆盖率更准

两篇文章可能都是 90% 覆盖率，但一篇的未知词集中在高频核心词（严重影响理解），另一篇集中在低频专有名词（基本不影响）。所以不仅要看覆盖率，还要看**未知词的频率排名**。

### 词汇的"临界质量"

研究表明：
- 前 **1000** 高频词覆盖日常文本 ~75%
- 前 **2000** 词覆盖 ~80%
- 前 **5000** 词覆盖 ~90%
- 要达到"舒适阅读"需要 **8000-9000** 词

中间这段（2000→8000）是最难熬的，也是最需要精准推荐来维持动力的阶段。

## 设想

### 词汇画像：从哪里知道用户认识哪些词

**显式信号**：用户主动学习的卡片（Deck 中的 Card）。

**隐式信号**（更有价值）：
- 用户查了哪些词的翻译 → 不查的词大概率认识
- 词典查询历史 → 查过的词 = 可能不认识（或者刚学）
- 阅读行为 → 在文本中停留/查阅的位置暴露盲区

**冷启动**：用词汇量测试（如 Yes/No Test：给用户一堆词混入假词，勾选认识的）快速估算词汇量区间，映射到频率列表的前 N 个词。

### 限定领域：降维的关键

与其对全部英语（17 万词）建模，不如让用户选择一个感兴趣的**内容类型**，把词汇空间缩小到一个可控的范围。

**为什么限定领域是质变而非量变**：

1. **词汇空间急剧缩小**：泛英语需要覆盖 ~17000 词才能达到舒适阅读，但限定到一个类型（如言情、科技、游戏）后，有效词汇空间可能只有 3000-4000 个词。建模难度直接降了一个数量级。

2. **词汇重复率高**：泛领域文本的问题是同一篇的生词在下一篇里可能再也不出现。但在同一类型内，核心词汇会反复出现，用户的词汇模型在 5-10 篇阅读后就能收敛到比较准的状态。

3. **句法难度被锁定**：同一类型的句式模式相对固定（如言情 = 对话 + 内心独白 + 场景描写），不会突然出现跨类型的句法跨度。词汇覆盖率几乎是难度的唯一变量，非词汇因素干扰很小。

4. **领域专用词会被通用词频表严重误导**："blush" 在通用词频表可能排 8000+，但在言情小说里第一页就可能出现三次。所以必须为每个类型建立**领域专属词频表**，而不能直接套用通用词频排名。

**用户选择类型**：让用户主动选择感兴趣的类型（言情、科技博客、游戏攻略、历史、烹饪等），选得越细，词汇空间越小，推荐越准。

### 文本匹配：怎么找到恰好合适的文章

1. 让用户选择感兴趣的内容类型
2. 为每个类型维护一个**领域专属内容库**，并用该类型的语料生成**专属词频表**
3. 对每篇内容预计算**词汇频率指纹**——每个词原形出现了多少次
4. 用用户在该类型下的词汇画像计算覆盖率，推荐 90-98% 区间的文本
5. 用户阅读后更新词汇模型，形成反馈循环

### 从行为中学习：不需要手动标记

理想状态是系统自动推断词汇掌握程度：

- 查过的词 → 可能不熟
- 在阅读中反复遇到且从未查询的词 → 大概率认识
- 长期未出现且未查询的词 → 逐渐从"已知"降级为"模糊"
- 最终把所有信号融合为一个"掌握度评分"

### 阅读与复习的闭环

- 阅读中遇到的未知词 → 自动推荐加入学习
- 新学的词汇 → 实时影响推荐文本的难度
- 阅读中嵌入微型复习 → 遇到"学习中"的词时弹出提示
- 形成循环：学 → 用（阅读）→ 发现盲区 → 复习 → 再用s
